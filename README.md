## Что это

Цифровой двойник участка метро — это **концепция** интерактивной веб-платформы для мониторинга и прогнозирования состояния подземной инфраструктуры Санкт-Петербургского метрополитена.

Система объединяет:
- **3D-визуализацию** тоннелей метро, геологической структуры грунта и наземных зданий
- **Мониторинг в реальном времени** данных с датчиков (осадки грунта/зданий, деформации обделки метро)
- **ML-прогнозирование** развития деформаций
- **Оценку рисков** с цветовой индикацией статуса (зеленый/желтый/красный)
- **Автоматические рекомендации** по необходимым действиям

## Пример использования

**Сценарий:** Инженер проверяет состояние участка метро в начале рабочего дня

1. **Заход в систему**
   - Инженер открывает веб-платформу и авторизуется
   - Выбирает интересующий участок метро

2. **Визуализация**
   - Видит **3D-срез тоннеля** с цветовой индикацией состояния
   - Переключается между слоями: метро → грунт → здания сверху

3. **Мониторинг данных**
   - Просматривает **график осадок дома №15** над участком метро
   - Данные отображаются за последние 30 дней
   - Видит текущее значение осадки: **8.2 мм** (при допустимом 10 мм)

4. **Прогноз**
   - ML-модель показывает **прогноз на месяц вперед**
   - Прогнозируемая осадка через месяц: **9.8 мм**

5. **Оценка рисков**
   - Система показывает предупреждение: **"Риск превышения допустимых осадок — 25%"**
   - Статус объекта: **🟡 Желтый** (требуется внимание)

6. **Рекомендации**
   - Увеличить частоту измерений до 2 раз в неделю
   - Запланировать обследование здания №15 в течение 14 дней
   - Подготовить план укрепительных мероприятий на случай ухудшения ситуации

7. **Действия инженера**
   - Создает наряд на дополнительные измерения
   - Формирует отчет для руководства
   - Ставит объект на усиленный контроль

## Основные функции

### 1. 3D-визуализация инфраструктуры

**Варианты реализации для диплома:**

**A) Упрощенная схематичная модель (Three.js)** — рекомендуется

- Здания: параллелепипеды с цветовой индикацией риска
- Грунт: слои с текстурами
- Датчики: маркеры на 3D-сцене
- ⏱️ Время разработки: 1-2 недели
- ✅ Достаточно для демонстрации концепции

**B) Интеграция с Plaxis/Abaqus** — для продвинутого диплома

- Визуализация в Three.js
- ⏱️ Время: 3-4 недели
- 🏆 Значительное преимущество (публикация, отличная оценка)

**Возможности визуализации:**
- Интерактивная навигация (масштабирование, вращение)
- Переключение слоев (метро ↔ грунт ↔ здания)
- Клик по объекту → показ детальной информации
- Цветовая индикация статусов (🟢🟡🔴)

**Индикация на 3D-модели:**
- Цветовое выделение проблемных зон
- Отображение расположения датчиков
- Визуализация зон риска

### 3. Прогнозирование развития деформаций (ML)

**Универсальная модель для всех станций:**

Система использует **единую базовую модель** с параметризацией под конкретную станцию:
```python
# Инженер вводит параметры станции
station = {
    'name': 'Адмиралтейская',
    'depth': 86,  # глубина тоннеля, м
    'soil_type': 'cambrian_clay',
    'building_distance': 20  # расстояние до здания, м
}

# Система автоматически делает прогноз
forecast = model.predict(station)
```

**Источники данных для обучения ML:**
- ✅ **Синтетические** (сгенерированные по нормативам СП 22.13330.2016)
- ✅ **Из публикаций** (КиберЛенинка, диссертации)
- ✅ **Из Plaxis/Abaqus** (FEM-моделирование)
- 🟡 **Реальные** (если получится от Ленметрогипротранс/Горного университета)

📖 См. [CRITICAL_ANALYSIS.md](CRITICAL_ANALYSIS.md#много-ли-данных-нужно) про источники данных

**Возможности:**
- Прогноз развития осадок на 1-3 месяца вперед
- Прогноз деформаций обделки метро
- Учет сезонных факторов (уровень грунтовых вод, температура)
- Учет внешних воздействий (строительство, вибрации)
- Доверительные интервалы прогноза
- Сценарное моделирование ("что если...")

**Типы прогнозов:**
- **Краткосрочный**: 7-14 дней (точность ~90%)
- **Среднесрочный**: 1-3 месяца (точность ~75%)
- **Сценарный**: при различных условиях (например, при строительстве нового здания)

**Алгоритмы:**
- Нейронные сети для временных рядов (LSTM) — основа
- Градиентный бустинг (XGBoost) — для классификации рисков
- Prophet (Facebook) — для учета сезонности
- Autoencoder — для детекции аномалий
- Ансамблевое объединение моделей

### 4. Система оценки рисков и рекомендаций

**Цветовая индикация статуса:**

🟢 **Зеленый — Норма**
- Все параметры в пределах допустимых значений
- Прогноз стабилен
- **Риск < 10%**
- Действия: стандартный режим мониторинга

🟡 **Желтый — Внимание**
- Параметры приближаются к пороговым значениям
- Нетипичные изменения трендов
- **Риск 10-50%**
- Действия: усиленный контроль, планирование мероприятий

🔴 **Красный — Критично**
- Превышение допустимых значений
- Быстрое развитие деформаций
- **Риск > 50%**
- Действия: немедленное реагирование, возможно ограничение движения

**Метрики рисков:**
- Риск превышения допустимых осадок (%)
- Риск критических деформаций обделки (%)
- Общий индекс безопасности участка (0-100)
- Время до критического состояния (дни)

**Автоматические рекомендации:**
- Увеличение частоты измерений
- Проведение дополнительных обследований
- Планирование ремонтных работ
- Ограничение скорости движения поездов
- Укрепление конструкций
- Уведомление контролирующих органов

**Система уведомлений:**
- Push-уведомления при изменении статуса
- Email-рассылки для ответственных лиц
- SMS в критических ситуациях
- Интеграция с диспетчерскими системами
